# 업로드한 이미지를 포스트에서 사용하는 예시

블로그를 운영하다 보면 글 중간에 캡처 이미지나 예시 화면을 넣어야 하는 경우가 많다.  
이번 글은 저장소에 업로드한 이미지를 실제 포스트에서 어떻게 사용하는지 정리하는 예시용 글이다.

이미지는 현재 `assets/images/` 아래에 저장되어 있고, 본문에서는 `relative_url` 필터를 사용해 불러왔다.

## 1-1-002

첫 번째 이미지는 프로젝트 구조와 파일 배치를 설명할 때 사용할 수 있는 예시 화면이다.

![1-1-002]({{ '/assets/images/1-1-002.jpg' | relative_url }})

이런 종류의 이미지는 "현재 어떤 파일이 어디에 있는지"를 독자에게 빠르게 보여줄 때 유용하다.

## 1-1-003

두 번째 이미지는 바로 다음 단계나 세부 설정 화면을 이어서 설명할 때 적합하다.

![1-1-003]({{ '/assets/images/1-1-003.jpg' | relative_url }})

튜토리얼 글에서는 이미지 한 장보다, 흐름이 이어지는 여러 장의 캡처가 훨씬 이해를 돕는다.

## 1-1-004

세 번째 이미지는 구성 파일이나 메뉴 이동 과정을 설명하는 중간 장면으로 활용할 수 있다.

![1-1-004]({{ '/assets/images/1-1-004.jpg' | relative_url }})

설정형 글에서는 "무엇을 클릭해야 하는지"를 텍스트만으로 설명하는 것보다 이미지와 함께 보여주는 편이 훨씬 친절하다.

## 1-1-005

네 번째 이미지는 실습 과정 중 특정 상태를 기록하는 용도로 넣기 좋다.

![1-1-005]({{ '/assets/images/1-1-005.jpg' | relative_url }})

예를 들어 저장소 연결 직후, 워크플로 실행 직전, 또는 설정 저장 이후의 화면을 남길 때 잘 어울린다.

## 1-1-006

다섯 번째 이미지는 결과 확인 단계에서 사용하기 좋은 예시다.

![1-1-006]({{ '/assets/images/1-1-006.jpg' | relative_url }})

특히 블로그 구축 글에서는 "설정" 장면뿐 아니라 "정상 동작 결과" 장면도 함께 넣어주는 것이 중요하다.

## 1-1-008

마지막 이미지는 전체 흐름을 마무리하거나, 추가 설명이 필요한 화면을 보완하는 데 사용할 수 있다.

![1-1-008]({{ '/assets/images/1-1-008.jpg' | relative_url }})

이처럼 여러 장의 이미지를 한 포스트 안에 순서대로 배치하면 따라 하기 글의 완성도가 높아진다.

## 포스트에서 이미지 넣는 방법

프로젝트 페이지 방식의 GitHub Pages에서는 `baseurl`이 붙기 때문에, 이미지 경로를 직접 쓰기보다 아래처럼 `relative_url`을 사용하는 편이 안전하다.

```md
![이미지 설명]({{ '/assets/images/1-1-002.jpg' | relative_url }})
```

앞으로 이미지가 많아진다면 글별로 파일명을 더 명확하게 정리하거나, `assets/images/posts/` 아래에 시리즈별 폴더를 두는 방식도 고려할 수 있다.
