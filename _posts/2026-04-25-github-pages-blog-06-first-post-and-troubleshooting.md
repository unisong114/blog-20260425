# GitHub Pages 블로그 만들기 6. 첫 글 작성과 자주 만나는 문제

여기까지 왔다면 저장소도 있고, 테마도 적용했고, 배포 워크플로도 준비된 상태다.  
이제 정말 블로그답게 만드는 마지막 단계는 첫 글을 올리고, 자주 마주치는 문제를 피하는 것이다.

이번 글에서는 첫 포스트 작성 방법과 세팅 과정에서 자주 만나는 오류들을 함께 정리해보겠다.

## 첫 글은 어떻게 만들까

Jekyll에서는 `_posts` 폴더 아래에 Markdown 파일을 만들면 된다.

파일 이름은 아래 형식을 따른다.

```text
YYYY-MM-DD-title.md
```

예를 들면:

```text
2026-04-25-my-first-post.md
```

이 파일의 상단에는 front matter를 넣는다.

```md
---
title: "내 첫 글"
date: 2026-04-25 18:00:00 +0900
categories: [blog]
tags: [first-post, jekyll]
---
```

그 아래부터는 일반적인 Markdown 문법으로 본문을 작성하면 된다.

## 처음에는 어떤 글을 쓰면 좋을까

블로그 첫 글은 대단한 주제일 필요가 없다.  
오히려 아래 같은 글이 시작하기 좋다.

- 왜 이 블로그를 만들었는지
- 어떤 주제를 다룰 예정인지
- 현재 기술 스택이나 관심사
- 기록을 어떤 방식으로 남길 것인지

첫 글의 목적은 완성도보다 방향을 만드는 데 있다.

## 메뉴와 페이지를 함께 정리하면 좋다

포스트만 쌓기 시작하면 방문자는 이 사이트가 어떤 성격인지 파악하기 어렵다.  
그래서 최소한 아래 페이지 정도는 초반에 마련해두는 편이 좋다.

- `Home`
- `About`
- `Posts` 또는 글 목록 링크

`Just the Docs` 계열 테마는 이런 정보 구조를 비교적 잘 보여준다.

## 자주 만나는 문제 1. 404가 뜨는 경우

배포가 끝났는데 접속 시 404가 나온다면 먼저 주소를 확인한다.

프로젝트 페이지라면 보통 아래 형태다.

```text
https://username.github.io/repository-name/
```

루트 주소만 열고 있지는 않은지 먼저 확인하는 것이 좋다.

## 자주 만나는 문제 2. 화면은 뜨는데 CSS가 깨지는 경우

이 경우는 대부분 `_config.yml`의 `baseurl` 문제다.

예를 들어 저장소 이름이 `blog-20260425`라면 다음처럼 맞춰야 한다.

```yml
url: https://unisong114.github.io
baseurl: /blog-20260425
```

이 값이 틀리면 정적 리소스 경로가 어긋난다.

## 자주 만나는 문제 3. Actions에서 Pages 관련 오류가 나는 경우

대표적으로 아래 상황을 먼저 점검하면 된다.

- `Settings > Pages > Source`가 `GitHub Actions`인지
- 워크플로에 `pages: write`, `id-token: write` 권한이 있는지
- `build`는 성공하고 `deploy`에서만 실패하는지

오류는 대부분 로그를 보면 어느 단계에서 막혔는지 힌트가 나온다.

## 자주 만나는 문제 4. 로컬 실행이 되지 않는 경우

로컬에서 Jekyll을 띄우려면 Ruby와 Bundler가 필요하다.

보통은 아래 흐름으로 실행한다.

```bash
bundle install
bundle exec jekyll serve
```

여기서 `bundle`이나 `ruby` 명령 자체가 없으면, 먼저 개발 환경 설치가 필요하다.

## 자주 만나는 문제 5. 글은 올렸는데 목록에 안 보이는 경우

아래 항목을 차례대로 확인해보면 된다.

- 파일이 `_posts` 폴더 아래에 있는지
- 파일 이름이 날짜 규칙을 따르는지
- front matter가 올바른지
- 날짜 형식이 잘못되지 않았는지

사소해 보이지만 이 네 가지에서 문제가 나는 경우가 정말 많다.

## 시리즈를 마무리하며

이번 시리즈는 GitHub에 저장소를 만들고, Jekyll 테마를 연결하고, GitHub Pages로 배포해 블로그를 운영하는 전체 흐름을 기준으로 정리했다.

정리하면 전체 과정은 아래 여섯 단계였다.

1. 저장소 만들기
2. 로컬 폴더와 연결하기
3. Jekyll 테마 적용하기
4. 필수 파일 구성하기
5. GitHub Actions로 배포하기
6. 첫 글 작성과 오류 점검하기

처음에는 단계가 많아 보여도, 한 번 흐름을 잡아두면 이후에는 글 쓰는 일 자체에 훨씬 집중할 수 있다.

이제부터는 세팅보다 운영의 시간이다.  
꾸준히 글을 올릴 수 있는 구조를 만들었다는 점에서, 사실 가장 중요한 단계는 이미 끝났다고 봐도 좋다.
