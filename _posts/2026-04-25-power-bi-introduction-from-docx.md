## Power BI란?

Power BI는 Microsoft의 데이터 분석 및 시각화 서비스다.  
다양한 데이터 원본에 연결해 데이터를 정리하고, 시각화하고, 분석한 뒤, 그 결과를 보고서와 대시보드 형태로 공유할 수 있게 해준다.

실무 관점에서 보면 Power BI는 단순한 차트 도구를 넘어선다.

- 여러 데이터 원본 연결
- 정리와 변환을 통한 분석 준비
- 대시보드와 보고서를 통한 인사이트 도출
- 웹과 모바일 환경에서의 공유와 활용

즉, 데이터를 단순히 보는 수준이 아니라 의사결정에 활용할 수 있는 흐름 전체를 지원하는 도구라고 보는 편이 자연스럽다.

## Power BI의 세 가지 핵심 구성 요소

Power BI는 크게 세 가지 요소로 구성된다.

| 구성 요소 | 역할 | 사용 맥락 |
| --- | --- | --- |
| Power BI Desktop | 데이터 연결, 정리, 모델링, 보고서 작성 | 주로 제작자/분석가 |
| Power BI 서비스 | 보고서 게시, 대시보드 구성, 공유 | 웹 기반 협업과 배포 |
| Power BI Mobile | 게시된 콘텐츠 확인, 탐색 | 모바일 확인과 현장 활용 |

일반적인 작업 흐름은 다음과 같다.

1. Power BI Desktop에서 데이터를 연결하고 보고서를 작성한다.
2. 작성한 보고서를 Power BI 서비스에 게시한다.
3. 서비스에서 대시보드로 핵심 정보를 모아 공유한다.
4. 모바일 앱에서 필요한 내용을 빠르게 확인한다.

![Power BI Desktop, 서비스, 모바일의 관계]({{ '/assets/images/posts/power-bi-intro/image1.png' | relative_url }})

위 이미지는 Desktop에서 만든 보고서가 서비스와 모바일 활용으로 이어지는 전반적인 흐름을 한눈에 보여준다.

## Power BI에서 다루는 주요 콘텐츠

문서에서 설명한 핵심 콘텐츠를 표로 정리하면 아래와 같다.

| 항목 | 설명 |
| --- | --- |
| 의미 체계 모델 | 분석과 시각화를 위해 준비된 데이터 원본 |
| 시각화 | 막대, 원형, 분산형, 트리맵 등 데이터를 표현하는 시각적 개체 |
| 보고서 | 하나 이상의 페이지로 구성된 분석 화면 |
| 대시보드 | 여러 보고서의 핵심 정보를 한 화면에 모은 단일 캔버스 |
| 앱 | 대시보드와 보고서를 패키지처럼 묶어 배포하는 단위 |

이 구조를 이해하면 Power BI 화면을 처음 접했을 때도 "지금 내가 보고 있는 것이 보고서인지, 대시보드인지, 데이터 모델인지"를 훨씬 쉽게 구분할 수 있다.

## Power BI 시작과 계정 생성 흐름

처음 시작할 때는 공식 사이트에서 Power BI 서비스 진입 지점을 확인하게 된다.

![Power BI 공식 사이트 시작 화면]({{ '/assets/images/posts/power-bi-intro/image19.png' | relative_url }})

원문 문서에서는 계정 생성 흐름도 함께 설명한다. 핵심만 정리하면 아래와 같다.

1. `https://powerbi.microsoft.com`에 접속한다.
2. 우측 상단의 `무료로 시작`을 클릭한다.
3. 회사 또는 학교 메일 주소를 입력한다.
4. 새 계정 생성 흐름에 따라 필요한 정보를 입력한다.
5. 등록을 마치면 Power BI 홈 화면으로 이동한다.

여기서 중요한 조건이 하나 있다.  
원문 기준으로 Power BI 계정 생성에는 개인 메일이 아니라 회사 또는 학교 메일 주소가 필요하다. 문서에서도 `outlook.com`, `gmail.com`, `naver.com` 같은 상용 전자 메일 주소는 지원하지 않는다고 설명한다.

가입 화면 일부는 다음처럼 이어진다.

![Power BI 이메일 입력 화면]({{ '/assets/images/posts/power-bi-intro/image3.png' | relative_url }})

이후 단계의 원본 캡처에는 실제 메일 주소와 확인 코드 입력 화면이 포함되어 있어, 블로그 포스트에서는 개인정보 노출을 피하기 위해 해당 이미지는 생략했다.

## 샘플로 Power BI 화면 익히기

Power BI는 학습용 샘플을 제공하기 때문에, 처음에는 직접 데이터를 가져오기보다 샘플을 열어 구조를 익히는 편이 좋다.

문서에서는 Power BI 서비스의 `학습` 메뉴에서 샘플 보고서를 열어보는 방식을 안내한다.

![Power BI 학습 센터와 샘플 보고서]({{ '/assets/images/posts/power-bi-intro/image10.png' | relative_url }})

샘플을 열면 작업 영역에 대시보드, 보고서, 의미 체계 모델이 생성된다.  
이 단계에서 각각의 콘텐츠가 어떻게 연결되는지 확인해보면 개념 이해가 훨씬 빨라진다.

## 대시보드와 보고서 화면은 어떻게 다른가

문서에서는 생성된 샘플을 통해 대시보드와 보고서를 직접 비교하도록 안내한다.

먼저 대시보드는 핵심 지표를 요약해서 보여주는 한 장짜리 화면에 가깝다.

![Retail Analysis Sample 대시보드]({{ '/assets/images/posts/power-bi-intro/image12.png' | relative_url }})

여기서는 카드, 차트, 지도 등 여러 시각화가 한 화면에 모여 있고, 빠르게 전체 상황을 파악하는 데 초점이 맞춰져 있다.

반면 보고서는 좀 더 탐색적이다. 페이지를 이동하거나, 필터와 슬라이서를 조정하면서 데이터를 자세히 볼 수 있다.

![District Monthly Sales 보고서 페이지]({{ '/assets/images/posts/power-bi-intro/image14.png' | relative_url }})

![Store Sales Overview 보고서 페이지]({{ '/assets/images/posts/power-bi-intro/image15.png' | relative_url }})

원문에서 강조하는 포인트도 여기에 있다.

- 대시보드는 핵심 정보를 한눈에 본다.
- 보고서는 시각적 개체 간 상호작용을 통해 자세히 분석한다.
- 슬라이서와 필터를 조정해 조건별로 데이터를 살펴볼 수 있다.

## Power BI 라이선스 정리

원문은 라이선스를 사용자 기준으로 세 가지로 소개한다. 이를 표로 다시 정리하면 다음과 같다.

| 라이선스 | 특징 | 문서 기준 메모 |
| --- | --- | --- |
| 무료 계정 | 기본적인 시각적 분석과 개인 학습용 시작점 | Microsoft Fabric Free 환경에서 시작 가능 |
| Power BI Pro | 보고서 공유와 협업 기능 강화 | 사용자당 월 과금 |
| 사용자 단위 Power BI Premium | 더 큰 규모의 활용과 확장된 기능 | Pro보다 높은 기능 범위 |

![Power BI 라이선스 비교 화면]({{ '/assets/images/posts/power-bi-intro/image18.png' | relative_url }})

원문에는 이 외에도 조직 단위에서 고려하는 Power BI Premium과 Microsoft Fabric에 대한 설명이 함께 나온다.  
입문 단계에서는 우선 무료 계정, Pro, 사용자 단위 Premium의 차이만 이해해도 충분하다.
